## 为什么选择Gene插件系列？

Gene插件系列采用模块化设计，每个插件都专注于特定功能领域，同时保持高度的可扩展性和兼容性。这些插件共同构成了一个完整的服务器生态系统，能够满足各类服务器的需求。

主要应用场景：
  - **服务器管理**：提供全面的服务器管理工具
  - **玩家体验**：丰富的游戏内容和互动功能
  - **经济系统**：完整的经济体系支持
  - **安全防护**：多层次的安全保护机制

## 核心功能

以下是Gene插件系列的主要功能特点：

| 功能模块 | 描述 |
|---------|------|
| **服务器中间件** | 提供核心功能支持，包括数据存储、事件处理等 |
| **安全防护** | 防作弊、防攻击、玩家行为监控等安全功能 |
| **经济系统** | 多货币支持、交易系统、商店功能等 |
| **角色系统** | 玩家角色管理、权限控制、等级系统等 |
| **物品系统** | 自定义物品、物品种植、物品交易等 |
| **数据管理** | 玩家数据缓存、数据同步、备份恢复等 |
| **团队系统** | 团队管理、跨服组队、团队副本等 |

## 🏗️ 系统架构

Gene插件系列采用分层架构设计，确保各个模块间的低耦合和高内聚：

```
┌─────────────────────────────────────────┐
│              用户界面层                  │
│  (Web界面、游戏内GUI、命令系统)          │
└─────────────────────────────────────────┘
                    │
┌─────────────────────────────────────────┐
│             业务逻辑层                   │
│  (游戏逻辑、规则引擎、事件处理)          │
│  • Gene_Kether - 脚本引擎              │
│  • Gene_Roles - 角色管理                │
│  • Gene_Economy - 经济系统              │
│  • Gene_dungeonplus - 副本系统          │
└─────────────────────────────────────────┘
                    │
┌─────────────────────────────────────────┐
│             数据访问层                   │
│  (数据存储、缓存、同步)                 │
│  • Gene_PlayerDataCache - 数据缓存      │
│  • Gene_GlobalMail - 邮件系统            │
│  • Gene_ServerMiddleSystem - 中间件      │
└─────────────────────────────────────────┘
                    │
┌─────────────────────────────────────────┐
│             基础设施层                   │
│  (数据库、文件系统、安全防护)            │
│  • Gene_Shield - 安全防护                │
│  • MySQL/Redis - 数据存储                │
└─────────────────────────────────────────┘
```

### 核心依赖关系

- **Gene_ServerMiddleSystem** 是所有插件的基础，必须首先安装
- **Gene_PlayerDataCache** 为数据密集型插件提供缓存支持
- **Gene_Kether** 为需要脚本功能的插件提供脚本引擎
- **Gene_Shield** 提供全局安全防护

## 🚀 快速入门指南

### 环境要求
- **Minecraft版本**: 1.16.5+ (推荐最新版本)
- **Java版本**: Java 17+
- **服务器核心**: Paper/Spigot/Purpur
- **内存要求**: 最低4GB，推荐8GB+

### 基础安装步骤

1. **下载插件**
   ```
   从项目仓库下载所有需要的Gene插件JAR文件
   ```

2. **安装到服务器**
   ```
   将插件文件放入 plugins/ 目录
   ```

3. **启动服务器**
   ```
   启动服务器，插件会自动生成配置文件
   ```

4. **配置插件**
   ```
   根据服务器需求修改各插件的配置文件
   ```

5. **重启服务器**
   ```
   重启服务器以应用新配置
   ```

### 推荐安装顺序

我们建议按照以下顺序安装插件，以确保依赖关系正确：

1. **Gene_ServerMiddleSystem** ← 核心中间件
2. **Gene_Shield** ← 安全防护
3. **Gene_PlayerDataCache** ← 数据缓存
4. **Gene_Economy** ← 经济系统
5. **Gene_Roles** ← 角色管理
6. 其他功能插件 ← 根据需求选择

### 配置文件位置

所有插件的配置文件都位于 `plugins/Gene_[PluginName]/` 目录下。

## ❓ 常见问题解答

### 插件安装相关

**Q: 插件安装顺序有要求吗？**
A: 是的，建议按照文档中的推荐顺序安装。Gene_ServerMiddleSystem 必须首先安装，它是其他插件的基础。

**Q: 安装插件后服务器无法启动怎么办？**
A: 请检查：
1. Java版本是否为17+
2. 服务器核心是否支持（Paper/Spigot/Purpur）
3. 是否有依赖冲突
4. 查看服务器日志中的错误信息

**Q: 如何备份插件配置？**
A: 所有配置文件位于 `plugins/Gene_[PluginName]/` 目录，定期备份此目录即可。

### 性能相关

**Q: 插件会影响服务器性能吗？**
A: Gene插件经过优化设计，在正常使用情况下对性能影响很小。但大量使用Kether脚本可能需要更多内存。

**Q: 如何优化插件性能？**
A: 建议：
1. 使用Gene_PlayerDataCache进行数据缓存
2. 合理配置数据库连接池
3. 定期清理无用数据
4. 监控服务器资源使用情况

### 功能相关

**Q: 如何在不同服务器间同步玩家数据？**
A: 使用Gene_PlayerDataCache插件，它支持Redis作为数据缓存层，实现跨服数据同步。

**Q: 可以自定义经济系统吗？**
A: 是的，Gene_Economy支持多种货币类型和交易规则，可以根据服务器需求进行深度定制。

**Q: 安全防护功能覆盖哪些方面？**
A: Gene_Shield提供防作弊、防攻击、玩家行为监控等全方位安全防护。

### 故障排除

**Q: 插件命令不生效怎么办？**
A: 检查玩家权限设置，确保相关插件已正确加载且配置文件无误。

**Q: 数据不同步问题如何解决？**
A: 检查网络连接、数据库配置和缓存设置。重启相关服务通常可以解决大部分同步问题。
